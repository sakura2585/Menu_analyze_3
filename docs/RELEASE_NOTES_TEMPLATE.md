# Release Notes 模板

這份給 GitHub `Release` 直接貼上用。  
建議每次發版至少寫「本版重點 + 升級提醒」。

---

## 模板 A：完整版（建議）

```md
## 版本
- App: vX.Y.Z

## 本版重點
- 功能：
- 修正：
- UI/流程調整：

## 影響範圍
- 主標籤篩選：
- 匯出 / A4 PDF：
- 更新檢查：

## 升級提醒
- 是否需調整設定：否/是（說明）
- 相容性注意事項：無（或填寫）
```

---

## 模板 B：超短版（3 行）

```md
- 版本：vX.Y.Z
- 重點：新增/修正（1~2 句）
- 注意：是否需要調整設定（無則寫「無」）
```

---

## 範例：v1.0.1

```md
## 版本
- App: v1.0.1

## 本版重點
- 功能：新增主標籤篩選名單自定排序與排序記憶
- 修正：匯出與 A4 PDF 同步套用排序
- UI/流程調整：分量分計格式調整、數字紅色、工具列重新排版
- 功能：新增說明教學分頁與 GitHub Releases 更新檢查

## 影響範圍
- 主標籤篩選：排序規則可自訂且重開後保留
- 匯出 / A4 PDF：輸出順序與畫面一致
- 更新檢查：支援檢查最新 release

## 升級提醒
- 是否需調整設定：否
- 相容性注意事項：請先建立 GitHub Release，否則更新檢查可能顯示 404
```
