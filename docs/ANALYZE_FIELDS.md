# 分析欄位設定（`analyze_fields.json`）

同一套標籤規則（`order_note_parser._build_tags`）可套用在使用者勾選的欄位上；**未勾選的欄位不參與**關鍵字掃描、`#` 標籤與標籤庫補抓。

## 欄位說明

| 開關 | 對應資料 | 效果摘要 |
|------|----------|----------|
| **姓名欄** | Tab 第 2 欄 | 括號標籤（paren）、姓名側 NO 切塊、換餐/餐具等（若與方案/備註合併掃描時僅計入本欄片段）、hashtag／標籤庫（本欄） |
| **方案／品項欄** | Tab 第 3 欄 | 優惠（promo）、併入換餐/飯量/餐具等、hashtag／標籤庫（本欄）；**人數**僅在勾選欄位連成的文字中偵測 |
| **備註欄** | Tab 第 4 欄 | 備註側 NO 切塊、needs_review、hashtag／標籤庫（本欄） |
| **整列原文（輔助）** | 該列完整字串 | 額外從整列抽 `#`，並將整列納入標籤庫比對（利於表頭日期列或未標準 Tab 時補抓） |

序號欄位不列入選項（不提供意義）。

## UI

「輸入與分析」→ **分析欄位…** → 勾選後確定，寫入 `analyze_fields.json`。

## 預設

未建立設定檔時等同 **四項全開**。
